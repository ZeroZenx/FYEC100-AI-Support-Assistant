# Pilot Operations Runbook

## Purpose

This runbook gives the FYEC100 project team a shared operating procedure for
running a controlled Moodle pilot of the AI Academic Support Assistant.

It focuses on what the team should do before, during, and after pilot sessions,
including support routing, incident handling, academic integrity escalation, and
stop-pilot triggers.

## Admin Endpoints

- `/api/admin/pilot-operations-runbook`
- `/api/admin/pilot-operations-runbook.md`

The source record is `data/pilot-operations-runbook.json`.

## Review Guidance

- Review the runbook before each controlled Moodle pilot session.
- Confirm owners for lecturer, LMS, technical, infrastructure, accessibility,
  and project sponsor decisions.
- Export the Markdown runbook for project-team records.
- Do not record student names, student IDs, grades, or protected student record
  information in the runbook.

## Phase 2 Limitation

This is a pilot operations artifact. Production incident management,
accessibility accommodations, privacy review, and academic misconduct handling
must follow approved institutional procedures.
