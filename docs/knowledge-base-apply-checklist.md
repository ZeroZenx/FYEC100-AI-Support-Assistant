# Knowledge Base Apply Checklist

## Purpose

The knowledge base apply checklist documents the manual process for moving
approved draft wording into the live FYEC100 Markdown knowledge base.

It is intentionally not an auto-editor. The live file
`data/fyec100-knowledge-base.md` should only be changed after content-owner
review and a deliberate manual edit.

## Source File

```text
data/kb-apply-checklist.json
```

## Admin Endpoints

```text
/api/admin/knowledge-base/apply-checklist
/api/admin/knowledge-base/apply-checklist.md
```

## Required Review Steps

- Pre-apply review
- Content-owner approval
- Manual Markdown edit
- Guardrail review
- Smoke test
- Release note update

## Governance Boundary

This checklist supports auditability for Phase 2 pilot content updates. It does
not replace lecturer/content-owner approval, official academic policy, or a
future production content management system.
