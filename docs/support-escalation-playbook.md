# Support Escalation Playbook

Phase 2 includes an admin support playbook for routing common pilot issues.

## Admin Endpoint

```text
/api/admin/support-playbook
```

## Escalation Categories

- Lecturer follow-up
- LMS navigation or access
- Knowledge base update
- Technical or AI provider issue
- Academic integrity concern

## How To Use

During pilot review:

1. Check the feedback queue in `/admin`.
2. Match repeated issues to a playbook category.
3. Assign the listed owner.
4. Record knowledge base updates that require lecturer or content-owner review.
5. Pause or stop the pilot if a high-priority technical, privacy, or academic integrity issue cannot be resolved quickly.

This playbook is not a ticketing system. It is a Phase 2 support scaffold for controlled pilot governance.
